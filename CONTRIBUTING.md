# Contributing

## Branch/PR hygiene
- Keep PRs small and single-purpose.
- Link each PR to an issue when possible.
- Include reproduction commands for experiments and figures.

## Code conventions
- Prefer pure functions in `src/collatz/`.
- Put runnable entrypoints in `scripts/`.
- Add tests for behavior changes.

## Paper conventions
- Keep notation centralized in `paper/sections/notation.tex`.
- Figures must be produced by scripts.
- Add citations for claims about prior work.
